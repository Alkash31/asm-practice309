Коротко: int 0x80 — це програмне переривання в Linux (x86, 32-bit), через яке програма звертається до ядра ОС, тобто робить system call.

Тепер по кроках 

Що означає int

int = interrupt (переривання).

Команда:

int 0x80

каже процесору:

«Зупини виконання моєї програми і передай керування операційній системі».

0x80 — це номер переривання, який у Linux зарезервований саме для системних викликів.

Навіщо це потрібно

Звичайна програма не має прямого доступу до:

виводу на екран

файлів

памʼяті інших процесів

завершення процесу

Тому вона просить ядро зробити це за неї.

int 0x80 — це спосіб сказати:

«Ядро, зроби щось для мене»

Як ядро розуміє, ЩО саме зробити

Перед int 0x80 ти кладеш значення в регістри:

Регістр	Значення
eax	номер системного виклику
ebx	1-й аргумент
ecx	2-й аргумент
edx	3-й аргумент
Приклад з твого коду (вивід тексту)
mov eax, 4      ; sys_write
mov ebx, 1      ; stdout
mov ecx, msg    ; адреса рядка
mov edx, len    ; довжина
int 0x80

Це означає:

«Ядро, виклич write(1, msg, len)»

Завершення програми
mov eax, 1      ; sys_exit
int 0x80

Тобто:

«Ядро, закрий мій процес»

Важливо 

int 0x80 працює тільки в 32-бітному Linux

В 64-бітних системах використовують syscall, а не int 0x80

int 0x80 повільніший і вважається застарілим

Дуже коротка аналогія

- int 0x80 — це як кнопка дзвінка до ядра
Ти:

пишеш, що хочеш (у регістри)

дзвониш (int 0x80)

ядро робить роботу і повертається назад
